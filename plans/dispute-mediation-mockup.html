Reference — Status Workflow & Resolution Types

Status Flow

open — Filed, awaiting admin triage
|
under_review — Admin claimed, reviewing
|
awaiting_response — Admin needs more info
|
resolved — Final resolution issued
or
closed — Withdrawn/duplicate

Resolution Types

full_refundStripe refund of full amount
partial_refundStripe refund of specified amount
creditManual credit (see #400)
warning_ownerDocumented warning
warning_sitterDocumented warning
ban_ownerSets approval_status=banned
ban_sitterSets approval_status=banned
no_actionClose without action

Design decisions:

  • Purple accent for disputes (distinct from red incidents, emerald bookings)
  • Disputes build on incidents — incident link is optional, but all booking incidents shown as context
  • One active dispute per booking (partial unique index)
  • Admin claims disputes to prevent duplicate work
  • Admin internal notes (is_admin_note) hidden from parties
  • 14-day window for completed bookings, no limit for active bookings
  • Stripe refunds executed at resolution time, not dispute filing
  • Ban cascading deferred — ban only prevents new bookings
